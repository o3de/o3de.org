Detailed Description

Describes a custom Action Set. All applications will have custom Action Sets because that's how developers define the gameplay I/O.

Member Data Documentation

◆ m_name

AZStd::string OpenXRVk::OpenXRActionSetDescriptor::m_name

This name must be unique across all Action Sets listed in an Action Sets Asset. The content of this string is limited to the characters listed here: https://registry.khronos.org/OpenXR/specs/1.0/html/xrspec.html#well-formed-path-strings

◆ m_priority

uint32_t OpenXRVk::OpenXRActionSetDescriptor::m_priority = 0

Higher values mean higher priority. The priority is used by the OpenXR runtime in case several action sets use identical action paths and the highest priority will win the event.
