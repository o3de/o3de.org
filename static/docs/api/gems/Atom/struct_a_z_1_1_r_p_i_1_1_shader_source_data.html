Detailed Description

This is a simple data structure that represents a .shader file. Provides configuration data about how to compile AZSL code.

◆ m_keepTempFolder

bool AZ::RPI::ShaderSourceData::m_keepTempFolder = false

Typically the AssetProcessor always removes the Temp folder when an asset compiles successfully. By setting this flag to true, the Temp folder used to compile this shader won't be deleted if the shader compiles successfully. Also, if the ShaderBuildArguments enables shader debug symbols, the Temp folder won't be removed so it becomes easier to debug shaders with tools like RenderDoc or Pix.
