◆ m_minusArguments

AZStd::string AZ::RPI::ShaderSourceData::SupervariantInfo::m_minusArguments

Opposite to @m_plusArguments.

  • MCPP Macro definition arguments - AZSLc arguments. Because there are global compilation arguments, this one is useful to remove some of those arguments in order to customize the compilation of a particular supervariant.

◆ m_name

AZ::Name AZ::RPI::ShaderSourceData::SupervariantInfo::m_name

Unique name of the supervariant. If left empty, the data refers to the default supervariant.

◆ m_plusArguments

AZStd::string AZ::RPI::ShaderSourceData::SupervariantInfo::m_plusArguments
  • MCPP Macro definition arguments + AZSLc arguments. These arguments are added after shader_global_build_options.json & m_compiler.m_azslcAdditionalFreeArguments. Arguments that start with "-D" are given to MCPP. Example: "-DMACRO1 -DMACRO2=3". all other arguments are given to AZSLc. Note the arguments are added in addition to the arguments in <GAME_PROJECT>/Config/shader_global_build_options.json
