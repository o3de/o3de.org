Detailed Description

template<class Key, AZStd::size_t FixedNumBuckets, AZStd::size_t FixedNumElements, class Hasher = AZStd::hash<Key>, class EqualKey = AZStd::equal_to<Key>>
class AZStd::fixed_unordered_set< Key, FixedNumBuckets, FixedNumElements, Hasher, EqualKey >

Fixed unordered set container is based on unordered set and complaint with CTR1 (6.2.4.3). This is an associative container, all Keys are unique. insert function will return false, if you try to add key that is in the set. It has the extensions in the hash_table class. No allocations will occur using this container. For optimal performance the provided FixedNumBuckets should be a prime number and about 1/3-1/4 of the FixedNumElements.

Check the unordered_set AZStdExamples.

Note
By default we don't have reverse iterators for unordered_xxx containers. This saves us 4 bytes per node and it's not used in the majority of cases. Reverse iterators are supported via the last template parameter.
