Detailed Description

template<class Key, class MappedType, AZStd::size_t FixedNumBuckets, AZStd::size_t FixedNumElements, class Hasher = AZStd::hash<Key>, class EqualKey = AZStd::equal_to<Key>>
class AZStd::fixed_unordered_multimap< Key, MappedType, FixedNumBuckets, FixedNumElements, Hasher, EqualKey >

Fixed unordered multi map container is based on unordered_multimap CTR1 (6.2.4.6) The only difference from the fixed_unordered_map is that we allow multiple entries with the same key. You can iterate over them, by checking if the key is the same. No allocations will occur using this container. For optimal performance the provided FixedNumBuckets should be a prime number and about 1/3-1/4 of the FixedNumElements.

Check the fixed_unordered_multimap AZStdExamples.
