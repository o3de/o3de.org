Detailed Description

template<class Key, class MappedType, class Hasher = AZStd::hash<Key>, class EqualKey = AZStd::equal_to<Key>, class Allocator = AZStd::allocator>
class AZStd::unordered_multimap< Key, MappedType, Hasher, EqualKey, Allocator >

Unordered multi map container is complaint with CTR1 (6.2.4.6) The only difference from the unordered_map is that we allow multiple entries with the same key. You can iterate over them, by checking if the key is the same.

Check the unordered_multimap AZStdExamples.
