Detailed Description

template<class Key, AZStd::size_t FixedNumBuckets, AZStd::size_t FixedNumElements, class Hasher = AZStd::hash<Key>, class EqualKey = AZStd::equal_to<Key>>
class AZStd::fixed_unordered_multiset< Key, FixedNumBuckets, FixedNumElements, Hasher, EqualKey >

Fixed unordered multi set container is based on unordered multiset and complaint with CTR1 (6.2.4.5) The only difference from the unordered_set is that we allow multiple entries with the same key. You can iterate over them, by checking if the key stays the same. No allocations will occur using this container. For optimal performance the provided FixedNumBuckets should be a prime number and about 1/3-1/4 of the FixedNumElements.

Check the unordered_multiset AZStdExamples.

Note
By default we don't have reverse iterators for unordered_xxx containers. This saves us 4 bytes per node and it's not used in the majority of cases. Reverse iterators are supported via the last template parameter.
